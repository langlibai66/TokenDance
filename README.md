# MomentCard

一个面向短视频信息流场景的 AI 卡片原型项目。

MomentCard 试图回答一个问题：如果 AI 不是一个需要主动打开和输入的工具，而是一张在信息流里“刷到即成立”的内容卡片，会是什么样？

本仓库提供一个适合 GitHub 展示的基础前端原型，重点演示：

- 短视频场景下的 AI 信息流卡片表达
- `16:9` 展示比例的基础端界面
- 轻交互按钮与多场景切换
- 零依赖、可直接打开运行的静态页面结构

## 项目预览

项目首页为一个基础展示原型页，包含：

- 顶部项目介绍区
- 一块 `16:9` 的短视频端展示区域
- 3 个可切换的 AI 卡片场景
- 适合 GitHub、作品集和概念验证展示的说明模块

默认场景包括：

1. 今晚吃什么
2. 周末去哪儿
3. 比赛前看点卡片

## 设计目标

### 1. 刷到即理解

卡片必须在第一眼就让用户知道：

- 为什么它会出现
- 它在解决什么问题
- 用户下一步可以点什么

### 2. 信息刚刚好

内容不追求复杂，而是聚焦“当前场景下最有用的一条建议”。

### 3. 轻互动，不打断

用户不需要跳出信息流，只需要做轻量反馈，例如：

- 换一个
- 我喜欢
- 更省钱
- 提醒我

### 4. 像内容，也像服务

它既保留信息流内容的快速消费特征，也具备 AI 服务的个性化能力。

## 当前实现

当前仓库是一个静态前端原型，包含以下文件：


## 产品背景

在短视频信息流产品中，用户多数时候并不是带着明确目标进入，而是在连续滑动中等待下一条内容带来兴趣、情绪或满足。

MomentCard 想探索一种新的内容形态：

> 当用户刷到它时，AI 已经提前理解了用户当下的状态，并把最合适的信息、建议和选择准备好。

它不是一个等待用户主动使用的 AI 工具，而是一张在信息流中自动发生价值的 AI 卡片。

## 示例场景

### 场景一：今晚吃什么

AI 根据时间、天气和用户偏好，直接生成一张轻量晚餐建议卡片。

卡片可以包含：

- 今日推荐
- 推荐理由
- 预算范围
- 替代方案
- 一键换一组

### 场景二：周末去哪儿

AI 根据天气、位置和用户兴趣，生成一个轻量半日出行建议。

卡片可以包含：

- 去哪儿
- 为什么适合今天
- 路线建议
- 时间预估
- 风格切换

### 场景三：比赛前看点

当用户关注的比赛即将开始时，AI 可以生成一张赛前摘要卡。

卡片可以包含：

- 开赛时间
- 阵容与核心看点
- 关键球员
- 一句话总结
